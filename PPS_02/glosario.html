Glosario

Glosario

  • Adversario: Entidad que ataca o es una amenaza para un sistema, que no está autorizada a acceder a la información o modificarla, o que trabaja para vencer las protecciones que se le otorgan a la información.
  • Amenaza: Todo elemento, circunstancia o acción que constituye una causa de riesgo para algún activo de información. Una amenaza, ya sea natural, accidental o intencionada, provocará incidentes de seguridad si el sistema presenta vulnerabilidades.
  • Comando curl. Comando disponible en la mayoría de los sistemas basados ​​en UNIX. Es una abreviatura de “Client URL”. Los comandos curl están diseñados para funcionar como una forma de verificar la conectividad a las URL. Son una gran herramienta para transferir datos.
  • Falla: Desviación del componente o del sistema respecto de lo esperado.
  • Hacker: Persona con grandes habilidades en el manejo de computadoras que investiga un sistema informático para avisar de los fallos y desarrollar técnicas de mejora. Sin embargo, tradicionalmente, este término ha sido utilizado para designar a los piratas informáticos, esta definición es actualmente cuestionada por los  profesionales de la ciberseguridad: "Persona que accede ilegalmente a sistemas informáticos ajenos para apropiárselos u obtener información secreta".
  • Inteligencia: Actividad y al conocimiento resultantes del proceso analítico a que es sometida la información obtenida, con la finalidad de optimizar los procesos necesarios para la toma de decisiones. 
  • LDAP (Lightweight Directory Access Protocol): Protocolo a nivel de aplicación que permite el acceso a un servicio de directorio ordenado y distribuido para buscar diversa información en un entorno de red.
  • Man-in-the-middle o tercer hombre interpuesto: Estrategia de ataque en la que el atacante intercepta una comunicación entre dos partes, sustituyendo el tráfico entre ambas a voluntad y controlando la comunicación.
  • Metadatos: En su definición más simple son datos que describen otros datos. Generalmente, un archivo, además de contener datos, tiene información adicional sobre su organización y estructura, como por ejemplo: fecha de creación o modificación, descripción, autor y resumen.
  • NIST (National Institute of Standards and Technology): El Instituto Nacional de Estándares y Tecnologías es una agencia de la Administración de Tecnología del Departamento de Comercio de los Estados Unidos. Su misión es promover la innovación y la competencia industrial mediante avances en meteorología, normas y tecnología de forma que mejoren la estabilidad económica y la calidad de vida.
  • No SQL (Not Only SQL). Sistema de gestión de base de datos que difieren del modelo de bases de datos relacionales. Este tipo de almacenamiento de datos relaja las restricciones inherentes al propio modelo relacional. Este tipo de bases de datos no suelen proveer garantías ACID, pero resultan mucho más eficientes y baratas para manipular datos sin tener necesariamente que adaptarlos a una estructura rígida. Son ideales para almacenar datos de manera masiva, lo que responde muy bien a la enorme riqueza de datos que genera la Web, y analizarlos sin seguir necesariamente unos estándares que no necesariamente se adaptan a ellos.
  • OSINT (Open Source Intelligence): Significa Inteligencia de Fuentes Abiertas. Se trata de un conjunto de técnicas y herramientas para recopilar información pública, relacionar los datos y procesarlos. Es decir, aplicarle análisis e inteligencia a la gran cantidad de información públicamente accesible en Internet con el objetivo de extraer conclusiones útiles para una investigación, una monitorización, una campaña de marketing, etc.
  • OWASP (Open Web Application Security Project): Organización sin fines de lucro y una red global que trabaja para la seguridad del software libre, en particular en la Web.
  • Phishing: Método mediante el cual se obtiene información personal de un usuario a través de sitios web ficticios, correo electrónico u otros mensajes. Esta información se emplea principalmente para fines ilegales. El término es una contracción de "password fishing" (pesca de contraseñas).
  • PII (Personally Identifiable Information): La información de identificación personal es cualquier dato que podría identificar potencialmente a un individuo específico. Cualquier información que puede ser utilizada para distinguir una persona de otra, y que puede ser usada para quitarle el anonimato a los datos anónimos..
  • Pruebas E2E: Probar flujos de ejecución en una aplicación de comienzo a fin. se prueban escenarios que combinan diferentes casos de uso/funcionalidades tal cual como lo haría un usuario.
  • Pruebas Fuzzy: El fuzzing o fuzz testing consiste en pruebas semiautomáticas o automáticas donde se proveen datos al azar, inválidos y no esperados en la sección de entrada de datos del software.
  • Riesgo: Es la posibilidad de que una amenaza se produzca, dando lugar a un ataque.
  • Sniffing: Técnica de vigilancia y recogida pasiva de comunicaciones de la red, descodifica los protocolos y examina los contenidos en busca de información de interés.
  • TouchID: Método seguro de autentificación para aplicaciones móviles. Se basa en el reconocimiento biométrico, es decir, el reconocimiento inequívoco de personas mediante aspectos físicos únicos, como las huellas dactilares, la retina o el iris.
  • Vector de ataque: Un agujero o falla presente en la aplicación que compromete la seguridad de la misma.
  • Vulnerabilidad: Deficiencias de seguridad de un sistema informático que pueden permitir el acceso a usuarios no autorizados y la realización de acciones maliciosas intencionadas.
