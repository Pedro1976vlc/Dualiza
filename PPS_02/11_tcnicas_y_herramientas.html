1.1.- Técnicas y herramientas

Las técnicas y herramientas más utilizadas para obtener la información en OSINT son:

  • Buscadores habituales. Una de las funciones principales es rastrear continuamente Internet obteniendo e indexando prácticamente todo lo que se encuentra en abierto, por ejemplo: un administrador de sistemas ha configurado una carpeta de Apache para que se pueda ver un listado de sus archivos, entre los que se encuentra un fichero SQL con la estructura de la base de datos. El buscador analizará la página indexando el listado de ficheros,  siendo éste accesible de forma pública  y pudiendo encontrar la estructura de la base de datos. Una situación similar ocurre con los videos y grabaciones realizadas por cámaras de videovigilancia, que son almacenadas en la nube, si las configuraciones de seguridad no se realizan de forma correcta. A la técnica para obtener información publicada sensible se le denomina “Google Hacking”, aunque puede recibir el nombre del buscador. Se hace uso de los operadores de búsqueda para encontrar la información que se persigue. A las expresiones que utilizan operadores avanzados para encontrar información interesante se les denomina Google Dork. En el siguiente enlace se comentan los principales operadores de búsqueda de Google:

https://ahrefs.com/blog/es/operadores-de-busqueda-avanzada-de-google/

  • Buscadores especializados: Estos buscadores facilitan los procesos de búsqueda de información, y entre ellos destacan:

- Shodan:  Permite realizar búsqueda de recursos concretos como WebCam, bases de datos…Existe servicio de pago con más funcionalidades.

- NameCHK: Comprueba si un usuario existe en más de 150 sitios web. En caso de robo de credenciales, es posible comprobar si ese usuario existe en otros servicios y probar con la contraseña robada, ya que muchas personas utilizan la misma contraseña en varios sitios.

  • Buscadores de información de personas. Existen varios, destacando Spokeo, aunque se centra en Estados Unidos. 
  • Herramientas extracción metadatos. De los archivos y ficheros se puede obtener mucha información como autor, fecha y hora de creación, software y versión del mismo utilizada o incluso la geolocalización en caso de fotografías. Existen múltiples herramientas para la obtención de metadatos, algunas de las más conocidas:

- Metagoofil. Permite la extracción de metadatos de documentos públicos; como archivos PDF, imágenes, o ficheros DOC, como son direcciones de correo electrónico del personal de una empresa, el software utilizado para la creación de los documentos. También da la posibilidad de analizar un sitio web completo.

- ExifTool. Similar al anterior, permitiendo también la modificación de los metadatos de los ficheros.

  • API Rest de las redes sociales. Otra opción muy útil para obtener información, aunque en los últimos tiempos ha dejado de ser pública. Además, es necesario disponer de cuenta en la red social y burlar las medidas de seguridad establecidas, como tokens para el acceso a la información.
  • Maltego: Software especializado en analizar las relaciones entre personas, dominios, cuentas en redes sociales o metadatos entre otros. Posee tres versiones para clientes y tres versiones para servidor, la versión CE es gratuita aunque con funcionalidades limitadas. Permite añadir otras fuentes como el buscador Shodan o People Mon y construir grafos de relaciones a partir de las fuentes seleccionadas.
  • Recon-ng. Similar a Maltego permite añadir módulos con diferentes motores de búsqueda para recabar información. Se ejecuta sobre la línea de comandos y posee incluso una tienda de módulos.
  • TheHarvester. Otro recolector de información de fuentes públicas. Similar al anterior.
  • Searchcode. En este caso, la fuente pública es el código fuente existente en repositorios, es posible realizar búsquedas para encontrar fallos en el mismo o datos sensibles que se han olvidado en el código, como comentarios con las claves de acceso a base de datos, o sentencias SQL que proporcionen información sobre la estructura de la base de datos.
  • OSINT Framework. Colección de recursos de fuentes abiertas, clasificados en función de la topología: como usuarios, correos electrónicos, imágenes y un largo etcétera.

Los metadatos son, en su definición más simple, datos que describen otros datos. Generalmente, un archivo, además de contener datos, tiene información adicional sobre su organización y estructura, como por ejemplo: fecha de creación o modificación, descripción, autor y resumen.

Protección frente a técnicas y herramientas de investigación en fuentes abiertas

Se ha de garantizar que las aplicaciones y sistemas que se desarrollen no poseen puntos de fuga de información, protegiéndose de las aplicaciones anteriores. Algunas de las acciones a realizar son:

  • Eliminar los metadatos de forma automática al subirlos al servidor.
  • Realizar búsquedas de correos expuestos en Internet del dominio de la organización.
  • Comprobar la configuración de los servidores para ver que solo se exponen al público las carpetas y ficheros  necesarios y no otros.
  • Configurar correctamente las condiciones de indexación en los buscadores.
  • Utilizar las herramientas OSINT para comprobar qué información se ofrece de forma pública.
  • Revisar la configuración de las redes sociales para proteger la información privada.
  • Comprobar de forma exhaustiva todas las APIS disponibles comprobando si son públicas o privadas y en caso de las públicas asegurarse que la información que muestra es la que se desea.
