Introducción

En la unidad anterior se vio cómo llevar a cabo la ejecución de los diferentes tipos de pruebas sobre las aplicaciones software. Se destacó la importancia que tiene el proceso de pruebas para garantizar la consecución de un software de calidad.

En esta unidad se presentan los diferentes métodos y recursos que sirven de ayuda a los desarrolladores para determinar el nivel de seguridad requerido por aplicaciones web y móviles, para  conseguir su puesta en producción de un modo seguro.

En primer lugar, se resalta la importancia del empleo de técnicas de inteligencia basadas en fuentes abiertas para el desarrollo seguro de las aplicaciones. Se presentan los distintos tipos de herramientas que se pueden emplear para investigar y extraer información con  la que poder valorar el nivel seguridad de las aplicaciones desarrolladas.

Diversas organizaciones gubernamentales, comerciales o sin ánimo de lucro, elaboran clasificaciones para los diferentes tipos de vulnerabilidades que se pueden encontrar en las aplicaciones informáticas.. En estas clasificaciones se encuentran las vulnerabilidades más populares y habituales como: la inyección SQL, la secuencia de comandos en sitios cruzados (XSS), la falsificación de peticiones en sitios cruzados (CSRF), o el empleo de algoritmos de cifrado débiles.

Entre las clasificaciones de vulnerabilidades, actualmente, destacan las publicadas por organizaciones como Open Web Application Security Project (OWASP) o la Corporación MITRE. 

La Fundación OWASP es una organización sin ánimo de lucro fundada en 2004 cuyo propósito principal es “Ser la comunidad global más próspera que impulse la visión y la evolución de la seguridad y protección en el software mundial”. OWASP realiza periódicamente la clasificación de los riesgos de seguridad más críticos para las aplicaciones web y móviles. Estos documentos son conocidos como OWASP Top 10 (OWASP Top Ten).

El principal objetivo del Estándar de Verificación de Seguridad (ASVS) de OWASP es proporcionar una guía normalizada para el desarrollo y mantenimiento de aplicaciones web y móvil,  que sirva de ayuda para establecer y verificar los requisitos de seguridad que estas deben cumplir, en función del nivel de seguridad  que deban implementar, para tratar y proteger adecuadamente la información que manejan.
