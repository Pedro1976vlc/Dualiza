3.1. - Estándares para medir la calidad del software.

Existen diferentes estándares de organizaciones internacionales para medir la seguridad del software, se tiene las normativa ISO que establece diferentes estándares sobre la seguridad como:

  • ISO/IEC 27001. Especifica los requisitos necesarios para establecer, implantar, mantener y mejorar un sistema de gestión de la seguridad de la información.

Para saber más

Vídeo sobre el estándar ISO 27001 para la seguridad de la información.

Descripción textual alternativa.

Explicación del estándar ISO 27001 que indica los requisitos para establecer, implantar, mantener y mejorar los sistemas de gestión de la seguridad de la información..

  • ISO/IEC 27002. Establece diferentes aspectos como las políticas de seguridad, gestión de activos, el control de acceso o las técnicas de cifrado entre otros.
  • ISO/IEC 27032. Este estándar se centra en     la seguridad en las redes,  la seguridad en Internet,  la seguridad de la Información y  la protección de las Infraestructuras Críticas para la Información.
  • ISO/IEC 27034: garantizar que las aplicaciones informáticas aseguren el nivel de seguridad requerido para el apoyo del Sistema de Gestión de Seguridad de la Información.
  • ISO 15408. Criterios de seguridad establecidos para medir la seguridad del software, aceptado por más de 22 países, también conocidos como “criterios comunes", estableciendo diferentes niveles en función del nivel de cumplimiento. Por ejemplo el nivel 3:

EAL3 (probado y verificado metódicamente): permite a un desarrollador alcanzar una máxima garantía de ingeniería de seguridad positiva en el estado de diseño sin la alteración substancial de prácticas de desarrollo válidas existentes. El análisis en este nivel se apoya en las pruebas de caja gris (grey box), la confirmación selectiva independiente de los resultados de las pruebas del desarrollador, y la evidencia de búsqueda de vulnerabilidades obvias del desarrollador. Además, se realizan controles del entorno de desarrollo y de gestión de configuración del producto.
