1.2.- Desarrollo seguro y fuentes abiertas.

Además de evitar la fuga de información, a la hora de crear software existen fuentes abiertas de diferentes organismos que se centran en los problemas de seguridad en el desarrollo de aplicaciones, las más conocidas son:

  • OWASP (Open Web Application Security Project): Proyecto de código abierto dedicado a determinar y combatir las causas que hacen que el software sea inseguro.
  • CWE (Common Weakness Enumeration): En castellano enumeración de debilidades comunes, proyecto comunitario para entender las fallas y el desarrollo de herramientas automatizadas para identificar, corregir y prevenir las fallas, se centra en el desarrollo, gestionado por la organización sin ánimo de lucro Mitre.
  • CVE (Common Vulnerabilities and Exposures): Acrónimo de  Vulnerabilidades y Exposiciones Comunes, similar al anterior, gestionado por la misma organización, aunque se centra más en software ya desarrollado, indicando la vulnerabilidad, las versiones del software afectadas y las posibles soluciones. Posee un buscador en el que se es posible buscar errores, bugs y fallas en el software y las librerías más utilizadas, la url es:

http://cve.mitre.org/cve/search_cve_list.html

Se puede también descargar la base de datos completa de la misma web.

Retroalimentación

Las vulnerabilidades de librerías y software se encuentran en CVE. Si existen vulnerabilidades en esa versión, al realizar la búsqueda. Sería necesario establecer un "sandbox" para sustituir la versión de la librería por otra más reciente, realizar los cambios en el software para esa nueva librería y establecer plan de pruebas para comprobar una integración correcta.
