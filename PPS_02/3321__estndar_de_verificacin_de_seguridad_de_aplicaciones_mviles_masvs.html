3.3.2.1. - Estándar de verificación de seguridad de aplicaciones móviles. (MASVS).

La guía de prueba establece diferentes tipos de pruebas que serán necesarias o no dependiendo del tipo de aplicación y el nivel de seguridad requerido, para establecer unos criterios comunes en cuanto al nivel de seguridad la fundación establece una serie de requerimientos que se han de cumplir en función de las pruebas (MSTG)

Estor requerimientos permiten conocer el grado de seguridad de una aplicación móvil en función del grado de cumplimiento de las pruebas. Establece 2 niveles (MASVS-L1 y MASVS-L2) de verificación de seguridad y otro adicional (MASVS-R) para la ingeniería inversa.

  • El nivel MASVS-L1: Se aplica a todas las aplicaciones, las implementación de este nivel tiene un coste económico y de usabilidad aceptable. Se centra en requerimientos relacionados con la calidad del código y manejo de datos sensibles. Debe existir un proceso de pruebas para verificar el nivel.
  • El nivel MASVS-L2: Se ha de usar en aplicaciones que manejen datos de especial protección, esto conlleva un aumento del coste económico en su implementación así como la pérdida de experiencia de uso por parte de los usuarios. En este nivel se ha de crear un modelo de amenazas y el desarrollo y la arquitectura la seguridad ha de ser un elemento clave, algunos ejemplos de aplicaciones de L2 son las relacionadas con la salud o con temas financieros. 
  • El nivel MASVS-R: Su función es evitar la ingeniería inversa y la manipulación de la aplicación para extraer código o datos sensibles. Se basa en las características del hardware y en técnicas de protección de software avanzadas.

El estándar actual se puede obtener en castellano y otros idiomas en la siguiente dirección:

https://github.com/OWASP/owasp-masvs/releases
