3.2. - Niveles de verificación.

OWASP establece en su Estándar de Verificación de Seguridad de Aplicaciones (ASVS) tres niveles diferentes:

  • ASVS L0 - Superficial. Este nivel no forma parte del estándar, de modo que no es un pre-requisito y puede saltarse directamente al nivel 1. Cada organización puede definir sus propios requisitos mínimos de seguridad. Indica que la aplicación posee algún tipo de verificación para estos requisitos de seguridad mínimos fijados por la propia organización. 
  • ASVS L1 - Oportunista. Pensado para todo tipo de software. No es necesario en principio el acceso al código y se definen para impedir las vulnerabilidades de OWASP Top Ten. Los ataques a estas aplicaciones suelen ser simples ya que el beneficio para los atacantes es poco.
  • ASVS L2 - Estándar. Aplicaciones que manejan datos sensibles y deben de cumplir con ciertos requisitos. Las aplicaciones manejan datos y comunicaciones personales como correos electrónicos o transacciones entre empresas o empresas y particulares. Los atacantes estudian la aplicaciones y poseen unos objetivos muy concretos, siendo capaces de emplear técnicas y herramientas avanzadas para descubrir y explotar vulnerabilidades de seguridad.
  • ASVS L3 - Avanzado. Es el nivel de verificación más exigente, trata datos confidenciales o maneja sistemas vitales. Reservado para aplicaciones militares, salud o infraestructuras entre otras, por ejemplo el control remoto de las válvulas y compuertas de una presa. 

Una aplicación en el nivel 3 de ASVS, requiere un análisis de mayor profundidad. Estas aplicaciones seguras deben ser claramente modulares (asegurando su resiliencia, escalabilidad, y correcta estructuración de sus capas de seguridad), y deben estar debidamente documentadas. 

Cada uno de estos niveles establece una lista de requerimientos de seguridad que pueden ser descritos en especificaciones de seguridad con funcionalidades y capacidades que pueden ser implementadas por los desarrolladores.
