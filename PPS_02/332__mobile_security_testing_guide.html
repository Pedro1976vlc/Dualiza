3.3.2. - Mobile Security Testing Guide.

Los dispositivos móviles almacenan datos personales muy importante para los usuarios, desde  los mensajes personales en programas de mensajería instantánea o los correos electrónicos hasta los datos bancarios, pasando por fotos personales. Esta información es una de las más deseadas por los hackers se hace necesario la creación de aplicaciones para móviles seguros, además de poder medir lo segura que es una aplicación con estándares de seguridad. OWASP establece   una serie de proyectos relacionados que permiten evaluar lo segura o insegura que es una aplicación.

  • Guía de desarrollo y testeo de seguridad en aplicaciones móviles.(MSTG). Guía de pruebas que define un conjunto de casos de pruebas junto con los procesos, técnicas y herramientas a utilizar para realizar dichas pruebas. Es una guía con un alto contenido técnico.
  • Estándar de verificación de seguridad de aplicaciones móviles (MASVS). Define una serie de requerimientos que las aplicaciones han de cumplir para conseguir ser certificadas con cierto nivel de seguridad, para cada requerimiento se define un caso de prueba en la guía de desarrollo y testeo (MSTG). 
  • Lista de verificación de seguridad de aplicaciones. Hoja de cálculo que permite realizar el seguimiento del proceso de evaluación de los requerimientos de una aplicación. Dispone de versión en castellano.

Guía de prueba de seguridad móvil(MSTG).

Esta guía define y analiza la  pruebas para dispositivos móviles. En especial en las aplicaciones móviles, describiendo un conjunto de requerimientos y pruebas exhaustivas que cubren los procesos, técnicas y herramientas utilizados durante una prueba de seguridad de aplicaciones para móviles, en el que se indican las vulnerabilidades típicas de las aplicaciones móviles, la metodología de pruebas sobre estas y las posibles soluciones, relacionándose con los requerimientos denominados MASVS, siendo MSTG  una aplicación técnica y tecnológica de estos, agrupados por tipo como:

  • Áreas privadas.
  • Almacenamiento local.
  • Comunicación con servicios externos, servicios web.
  • Autenticación y autorización.
  • Interacción con el sistema operativo.
  • Calidad de código.
  • Técnicas para evitar la manipulación e ingeniería inversa del código.

Descripción textual alternativa.

Análisis de los diferentes elementos para la verificación de aplicaciones móviles de la fundación OWASP, WSTG, MASVS y la hoja de cálculo para ralizar la evaluación.

También incluye capítulos con test de pruebas para las dos principales sistemas operativos móviles: Android e iOS y un apéndice en el que se indican las principales herramientas de test para estos sistemas operativos:

  • Distribuciones para pruebas como Androl4b o Adroid Tamer.
  • Frameworks de seguridad: AppMon para monitorización o MobSF para análisis.
  • Analizadores de código: Checkmarx, Fortity y Veracode.

En el caso de punto 3.2 referido a las pruebas de comunicaciones por red se indica:

Los principales ataque son los de "sniffing" y "man-in-the-middle".

Las principales las vulnerabilidades potenciales,  como la de interceptación de tráfico HTTP y HTTPS,  análisis de tráfico no HTTP, simulación de ataques "Man-in-the.Middle", envenenamiento de tablas ARP, o reenvío de puertos de puertos, junto con las herramientas para evaluarlas.

Los estándares de verificación de seguridad de aplicaciones móviles para las conexiones de red (MASVS). En este punto son 3. MSTG-NETWORK-1 y 2 relacionado con la codificación de datos en al red, los algoritmos a emplear, la comprobación de no exponer "endpoints" en texto plano o el análisis de tráfico enviado y MSTG-NETWORK-5 guía para aplicaciones críticas con niveles de verificación denominados "defensa en profundidad) que necesitan de implementar  controles de seguridad avanzados en operaciones críticas como validación con tarjeta de coordenadas en caso de pago en Internet.

Por último incluye un punto de referencia con el OWASP Mobile Top 10 relacionada con el capítulo, los MASVS, los CWE equivalentes, las herramientas a utilizar en esa guía de pruebas y otras referencias como los RFC de IETF o  el estándar de generación de certificados digitales .
