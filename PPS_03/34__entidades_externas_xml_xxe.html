3.4. - Entidades externas XML (XXE).

Las aplicaciones y, en particular servicios web basados en XML, o integraciones que utilizan XML, pueden ser vulnerables a este ataque sí:

  • La aplicación acepta XML directamente, carga XML desde fuentes no confiables o inserta datos no confiables en documentos XML. Por último, estos datos son analizados sintácticamente por un procesador XML.
  • Cualquiera de los procesadores XML utilizados en la aplicación o los servicios web basados en SOAP, poseen habilitadas las definiciones de tipo de documento (DTDs).
  • La aplicación utiliza SAML para el procesamiento de identidades. SAML utiliza XML para garantizar la identidad de los usuarios y puede ser vulnerable.
  • La aplicación utiliza SOAP en una versión previa a la 1.2 y, si las entidades XML son pasadas a la infraestructura SOAP, probablemente sea susceptible a ataques XXE.
  • Ser vulnerable a ataques XXE significa que probablemente la aplicación también es vulnerable a ataques de denegación de servicio, incluyendo el ataque Billion Laughs.

Billion Laughs Attack.:

Es un tipo de ataque de denegación-de-servicio (DoS) que apunta a parsers de documentos XML. Es también referido a como una bomba de XML o como un ataque de expansión de entidad exponencial.​ (https://es.wikipedia.org/wiki/Mil_millones_de_risas).

La experiencia del desarrollador resulta fundamental para evitar Ataques Externos XML para identificar y mitigar los defectos XXE.  Aparte de esto, prevenir XXE requiere:

  • De ser posible, utilizar formatos de datos sencillos como JSON y evitar la serialización de datos confidenciales.
  • Actualizar los procesadores y bibliotecas XML que utilizan la aplicación o el sistema subyacente.
  • Deshabilitar las entidades externas de XML y procesamiento DTD en todos los analizadores sintácticos XML en su aplicación, según se indica en la hoja de trucos para prevención de XXE de OWASP.
  • Validar y filtrar  la entrada de datos dañinos en el servidor. 
  • Verificar la funcionalidad de carga de archivos XML o XSL validar el XML entrante, usando validación XSD o similar.
  • Las herramientas SAST pueden ayudar a detectar XXE en el código fuente, aunque la revisión manual de código es la mejor alternativa en aplicaciones grandes y complejas.
  • Si estos controles no son posibles, considere usar parcheo virtual, gateways de seguridad de API, o Firewalls de de control de acceso.
