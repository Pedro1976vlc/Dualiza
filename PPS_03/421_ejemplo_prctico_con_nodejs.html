4.2.1.- Ejemplo práctico con NodeJs

Ejemplo de validación de datos recibidos en el servidor previos a la interacción con la base de datos, en este caso en NodeJs y framework web Express con servicios REST. Existen multitud de librerías en npm para la validación de datos recibidos:

w.npmjs.com/search?q=keywords%3Asanitization&ranking=popularity

Las más descargadas son:

  • Validator.
  • Express-Validator.
  • XSS.

En este ejemplo se utilizan la librerías express-validator en un servicio POST para el alta de la ficha personal de la clínica del ejemplo anterior.  El método encargado de recibir y procesar la petición es:

Validación en el servidor con Node

var almacen= new Array()
var regEx_sanityze = /<script>|<\/script>|<iframe>|<\/iframe>|javascript:|onload=|alert(1)|<|>|\"1=1\"|\'1\'=\'1\'|--,1=|SELECT/i
router.post('/',
  check('nombre').exists().withMessage('Campo nombre necesario').isLength({ min: 4, max: 50 }).withMessage('Longitud del nombre entre 10 y 50').customSanitizer((value) => {
    value = value.replace(regEx_sanityze, "")
    return value;
  }),
  check('apellidos').exists().withMessage('Campo nombre necesario').isLength({ min: 10, max: 110 }).withMessage('Longitud de los apellidos entre entre 10 y 110').customSanitizer((value) => {
    value = value.replace(regEx_sanityze, "")
    return value;
  }),
  check('direccion').exists().withMessage('Campo nombre necesario').isLength({ min: 10, max: 300 }).withMessage('Longitud de la dirección entre 10 y 300').customSanitizer((value) => {
    value = value.replace(regEx_sanityze, "")
    return value;
  }),
  check('peso').isInt({ min: 0, max: 300 }).withMessage('Peso fuera del rango entre 0 y 300').toInt(),
  check('altura').isInt({ min: 0, max: 250 }).withMessage('Altura fuera del rango entre 0 y 250').toInt(),
  check('edad').isInt({ min: 0, max: 110 }).withMessage('Edad fuera del rango entre 0 y 110').toInt(),
  (req, res) => {
    const errors = validationResult(req);
    if (!errors.isEmpty()) {
      return res.status(400).json({ errors: errors.array() });
    }
    var usuario = new  Usuario(req.body.nombre, req.body.apellidos, req.body.direccion, req.body.peso, req.body.altura, req.body.edad)
    almacen.push(usuario)
    res.json(usuario)

  }
);
router.get("/", (req,res)=>{
  res.json(almacen)
})
  • En la línea 2 se define una expresión regular para limpiar los parámetros de posibles inyecciones de código SQL o XSS (se necesita mejorar, pero es simplemente un ejemplo)
  • Desde la 4 a la 7 se realizan las comprobaciones necesarias para comprobar que los valores introducidos son válidos, además de eliminar los caracteres o cadenas que son sospechosos para el parámetro nombre. También se personalizan los mensajes de error.
  • En el cuerpo del método que procesa la petición, desde la línea 21 a la 23 se comprueba si las evaluaciones anteriores son correctas, en caso de no serlo se envía una respuesta con código de error y en el cuerpo los errores detectados.
  • Por último se crea el objeto Usuario y se almacena en un vector.
  • Se puede solicitar la lista de usuarios realizando una petición GET a esa misma URL

Reflexiona

En el ejemplo anterior se almacena en un simple vector. En caso de almacenar en una base de datos. ¿Qué precauciones tomar para evitar la inyección de SQL?

Para probar la validación se pueden utilizar diferentes herramientas como CURL, usar Selenium para la automatización de pruebas o como alternativa especializada en peticiones HTTP el software POSTMAN.
