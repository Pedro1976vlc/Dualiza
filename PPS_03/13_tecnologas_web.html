1.3. -Tecnologías Web.

Desde los inicios de la WWW en el que los clientes simplemente se encargaban de realizar la solicitud, interpretar el código HTML y presentar la web y la función del servidor era la de obtener el recurso estático (fichero) y enviarlo al cliente.

A medida que se comienza a utilizar y popularizar las páginas se empiezan a desarrollar tecnologías que ofrecen mayores funcionalidades tanto en el cliente como en el servidor, junto con estas tecnologías también aparecen vulnerabilidades en el desarrollo de las páginas web y aplicaciones. Para desarrollar aplicaciones web seguras se han de conocer las tecnologías implicadas, tanto de servidor como de cliente.

En el cliente:

HTML. Lenguaje de marcas de hipertexto.

Utiliza etiquetas con significado que el servidor envía al cliente, este lo recibe, interpreta y “pinta” en la pantalla. Existen diferentes versiones de HTML, en la actualidad se utiliza mayoritariamente HTML 5.

Un ejemplo sencillo de HTML.

<!DOCTYPE html><br><html><br><head><br><title>Primera página</title><br></head><br><body><br><p>Hola mundo.</p><br></body><br></html>



CSS

Los usuarios de las páginas o aplicaciones web esperan que estas sean atractivas visualmente, si bien HTML posee atributos y etiquetas que permiten modificar el aspecto visual de las aplicaciones, su mantenimiento cuando se tienen decenas o cientos de páginas en un sitio se hace imposible de mantener si se da el aspecto en cada etiqueta.

Para poder gestionar el aspecto de un sitio web de forma jerarquizada se utilizan las Hojas de Estilo en Cascada (Cascading Style Sheets), que utilizando selectores permiten modificar el aspecto de todas las páginas que hagan referencia a ese estilo. Un ejemplo de selector y atributos CSS:

body { 
background-color: lightblue; 
}
 p {
 font-family: verdana; font-size: 20px; 
}

Javascript.

Una vez la página web se encuentra en el cliente, se ha pintado en el navegador y aplicado los estilos es necesario realizar diferentes cálculos, operaciones o modificar el código HTML como puede ser añadir nuevas imágenes de forma dinámica a la página. Para estas y otras tareas se utiliza un lenguaje de programación denominado Javascript.

Javascript es uno de los lenguajes más utilizado en la actualidad, existiendo multitud de Frameworks para facilitar el desarrollo de aplicaciones. Un ejemplo de página web con JavaScript:

<!DOCTYPE html>
<html>
<body>
<h2>My First JavaScript</h2>
<button type="button"
onclick="document.getElementById('demo').innerHTML = Date()">
Click me to display Date and Time.</button>
<p id="demo"></p>
</body>
</html>

Debes conocer

El desarrollo de aplicaciones web del lado cliente se realiza actualmente utilizando la combinación de estas tres tecnologías principalmente, aunque han evolucionado a nuevos frameworks, herramientas y tecnologías como jQuery, Boostrap, Angular, SASS o Vue.

El cliente recibe el HTML, el CSS y Javascript de un equipo o conjunto de equipos denominados servidor o servidores. Inicialmente los elementos enviados eran simplemente ficheros estáticos, pero surgió la necesidad de a partir de los datos enviados por el cliente, por ejemplo el identificador único del cliente en Facebook generar un código html/css/javascript personalizado para la petición, por ejemplo al entrar en Instagram aparecen las fotos del usuario y no otras.

En cuanto las tecnologías de servidor.

Lenguajes de programación en el servidor.

Para realizar esta personalización se utilizan diferentes lenguajes de programación entre los que destacan:

  • PHP.
  • Java.(JSP).
  • Node.js (Basado en Javascript, pero en el servidor)
  • C#.

Estos lenguajes utilizan indicadores para que el servidor intérprete en el servidor lo necesario, por ejemplo en PHP se utilizan los delimitadores <? y ?> o <?php, un ejemplo de uso:

<!DOCTYPE html>
<html>
<body>

<?php
echo "My first PHP script!";
?>

</body>
</html>

Base de datos.

Las bases de datos son utilizadas en prácticamente todas las aplicaciones tanto de escritorio como web, aunque no forman parte directa de las aplicaciones se utilizan ampliamente para el almacenamiento de los datos de las aplicaciones web, pudiendo estar en equipos diferentes, además las bases de datos pueden ser utilizados por más de una aplicación, ya sea web o de escritorio.
