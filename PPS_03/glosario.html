Glosario

Glosario

  • Amenaza: Circunstancia desfavorable que puede ocurrir y que cuando sucede tiene consecuencias negativas sobre los activos provocando su indisponibilidad, funcionamiento incorrecto o pérdida de valor. Una amenaza puede tener causas naturales, ser accidental o intencionada. Si esta circunstancia desfavorable acontece a la vez que existe una vulnerabilidad o debilidad de los sistemas o aprovechando su existencia, puede derivar en un incidente de seguridad.
  • Balenceo de carga: Técnica que permite distribuir de forma dinámica, homogénea y en tiempo real, el tráfico entre dos puntos de una red, de manera que se evite la sobrecarga de una zona y el desaprovechamiento de otras.
  • CGI: Common Gateway Interface, programas que se ejecutan en el servidor , normalmente compilado que puede ser escrito en C, C++ o incluso Shell Script.
  • Clickjacking: Secuestro del clic. Es un ataque que engaña a un usuario para que haga clic en un elemento de la página web que es invisible o está disfrazado de otro elemento (HTML). Con ello se logra que los usuarios descarguen malware, visiten páginas web maliciosas, proporcionen credenciales o información confidencial, transfieran dinero o compren productos en línea sin darse cuenta.
  • Código heredado. Código fuente sin soporte técnico, del que no se dispone de documentación y su mantenimiento es costoso.
  • Cookie: Una cookie es un pequeño fichero que almacena información enviada por un sitio web y que se almacena en el equipo del usuario, de manera que el sitio web puede consultar la actividad previa del usuario. Sus principales funciones son:
    • Llevar el control de usuarios: cuando un usuario introduce su nombre de usuario y contraseña, se almacena una cookie para que no tenga que estar introduciéndolas para cada página del servidor.
    • Recabar información sobre los hábitos de navegación del usuario. Esto puede significar una ataque contra la privacidad de los usuarios y es por lo que hay que tener cuidado con ellas.

  • CSP (Content Security Policy). La Política de seguridad de contenido (CSP) es una capa adicional de seguridad que ayuda a detectar y mitigar ciertos tipos de ataques, incluidos Cross Site Scripting (XSS) y ataques de inyección de datos. Estos ataques se utilizan para todo, desde el robo de datos hasta la desfiguración del sitio y la distribución de malware. Para habilitar CSP, debe configurar su servidor web para que devuelva el encabezado HTTP Content-Security-Policy.

  • CSRF (Cross-site request forgery, falsificación de peticiones inter-sitio): Es una vulnerabilidad en un sitio web que permite a los atacantes forzar a las víctimas a ejecutar acciones sensibles en el sitio sin su conocimiento.
  • DBMS: Sistema gestor de base de datos (SGBD o DBMS), es un conjunto de programas que permiten el almacenamiento, modificación y extracción de la información en una base de datos.
  • Escalabilidad: Capacidad de adaptación y respuesta de un sistema con respecto al rendimiento del mismo a medida que aumentan de forma significativa el número de usuarios del mismo.
  • Extracción SSL: Reemplaza todas las peticiones HTTPS de una página web por HTTP para hacer un MITM entre el servidor y el cliente. El atacante es capaz de ver todo el tráfico en texto plano de la víctima.
  • Man-in-the-middle (MITM): Tercero interpuesto. Estrategia de ataque en la que el atacante intercepta una comunicación entre dos partes, sustituyendo el tráfico entre ambas a voluntad y controlando la comunicación.
  • MVC: Modelo, vista, controlador. Patrón software que separa la aplicación en modelo, vista y controlador, muy utilizado en el desarrollo web.
  • MVVC. MVVM: modelo–vista–modelo de vista, variación del MVC que intentan independizar aún más la presentación del resto del elementos de la aplicación.
  • ORM: Object–relational mapping. Software que permite transformar elementos de base de datos relacionales a objectos y viceversa.
  • P2P (Peer-to-peer). Red informática descentralizada de relación entre iguales, en la cual cualquiera de los miembros (puntos o nodos de la red) puede ocupar alternativamente el lugar de cliente o servidor en el proceso de intercambio. Permiten el intercambio directo de información, en cualquier formato, entre ordenadores interconectados, siendo habitualmente utilizados en internet para el intercambio de obras protegidas por el derecho de autor.
  • SEO: Profesionales que empleando diferentes técnicas permiten mejorar el posicionamiento en los buscadores, en especial Google.
  • Sistema federado: Conjunto de sistemas que actúan como uno solo.
  • Tolerancia a fallos: Capacidad introducida en un sistema para continuar funcionando correctamente; manteniendo un nivel especificado de rendimiento, pese a la presencia de fallos en los elementos que lo integran. Esta funcionalidad se logra mediante la presencia de elementos redundantes e independientes de proceso y almacenamiento.
  • Vulnerabilidad: Fallos o deficiencias de un programa que pueden permitir que un usuario no legítimo acceda a la información o lleve a cabo operaciones no permitidas de manera remota. Los agujeros de seguridad pueden ser aprovechadas por atacantes mediante exploits, para acceder a los sistemas con fines maliciosos.
