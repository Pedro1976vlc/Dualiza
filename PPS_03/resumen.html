Resumen

Esta unidad es extensa ya que en el desarrollo de aplicaciones web se utilizan diversos elementos, tecnologías y lenguajes.

  • Es fundamental para el desarrollo web y más concretamente para la seguridad el conocer en profundidad el protocolo HTTP, tratándose en el punto 1, junto con los elementos que intervienen.
  • Una vez se tiene los conocimientos de HTTP se describe el proceso y los roles para el desarrollo seguro, continuando con la lista OWAS para aplicaciones web, analizando los fundamentos, herramientas y caso de prueba de cada una de ellas.
  • Uno de los puntos más importante y que mayor causa de fallos de seguridad en las aplicaciones web ocasionan es la validación de los datos enviados al servidor y tratados por este. Se describen las técnicas para la validación tanto el el cliente como en el servidor, así como un ejemplo de validación en el lado cliente con el framework Angular y en el servidor con Nodejs y el framework Express.
  • El protocolo HTTP es un protocolo sin estado, por lo tanto es necesario conocer la identidad de quién realiza la petición cada vez que se realiza. La industria ha desarrollado diferentes tecnologías para ello, principalmente JWT y OAth.
  • Las contraseñas son uno de los recursos más importantes que se encuentran en el servidor, estas puede verse comprometidas por ataques de diferentes tipos, como por ejemplo inyección de SQL, en esta unidad se tratan las principales técnicas para proteger las contraseñas incluso en caso de fuga de información.
  • También se han desarrollado diferentes contramedidas para evitar ataques como fuerza bruta, XSS o ataqyes Men In The Middle, analizándose en el punto 8.
  • Por último se ha tratado el concepto Web Application Firewall que permite realizar filtrado, balanceo y modificación de peticiones HTTP.
