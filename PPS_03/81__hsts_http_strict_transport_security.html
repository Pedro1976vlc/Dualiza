8.1. - HSTS (HTTP Strict Transport Security)

Recomendación

Descripción  textual alternativa

Podtcast explicativo sobre el protocolo de Seguridad de Transporte Estricta para HTTP, conocido como HSTS. Además, se describe su implementación y uso con Wordpress.

El protocolo HSTS (HTTP Strict Transport Security) surgió con idea de reforzar la seguridad de los sitios web y solucionar algunos de los problemas detectados. Los detalles de su especificación se encuentran recogidos en la RFC 6797. Su principal objetivo es forzar que las comunicaciones entre el servidor y el cliente sean encriptadas y seguras.  Básicamente HSTS establece que todas las comunicaciones deben ser bajo un protocolo de comunicaciones seguro, como es HTTPS no estando permitidas las comunicaciones HTTP sencillas.

HSTS envía un mensaje al navegador web de usuario (conocido como UA o user agent) para indicarle que las comunicaciones deben ser encriptadas, en caso contrario estas deben ser revocadas y se mostrará un mensaje de error.

Directivas del encabezado HSTS

El encabezado HSTS consta de tres directivas:

max-age

Es la única directiva obligatoria, indica a los navegadores durante cuanto tiempo deben recordar que la comunicación con el sitio web es solo HTTPS. El valor de max-age viene expresado en segundos. Ejemplos de valores típicos son: 31536000 (1 año) o 63072000 (2 años). Si el max-age es 0, el navegador olvidará el sitio y lo tratará como uno nuevo en el próximo intento de conexión.

includeSubDomains

Es opcional e indica a los agentes de usuario que HTTPS también es obligatorio para todos los subdominios del dominio especificado.

preload

También es opcional, indica al navegador que el sitio web debe incluirse en la lista de precarga de HSTS.

Entre los sitios web que emplean HSTS destacan entre otros: Linkedin, Dropbox, Facebook, Twitter, Google o Yahoo.

Entre los navegadores que soportan HSTS destacan:

  • Google Chrome desde la versión 4.0.211.0.
  • Google Chrome para Android desde la versión 18.
  • Firefox y Firefox Mobile desde la versión 4.
  • Opera desde la versión 12.
  • Safari desde la versión 7.
  • Android Browser desde la versión 4.4 de Android.

Ventajas de HSTS

Ventajas de HSTS

Las principales ventajas que aporta HSTS son:

  • Todas las comunicaciones son seguras con HTTPS, incluyendo CSS y JavaScript.
  • Cualquier fallo que se produzca durante el establecimiento de la conexión segura provoca que esta se aborte.
  • Proporciona protección contra la extracción de SSL (SSL- stripping) en ataques man-in-the-middle.

Reemplaza todas las peticiones HTTPS de una página web por HTTP para hacer un MITM entre el servidor y el cliente. El atacante es capaz de ver todo el tráfico en texto plano de la víctima.
