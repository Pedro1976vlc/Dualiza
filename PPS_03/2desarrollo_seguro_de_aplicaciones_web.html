2.-Desarrollo seguro de aplicaciones web.

Las aplicaciones web se han convertido en unas de las más utilizadas en la actualidad, ya sea a través del navegador o utilizando librerías como Cordova o Ionic que hacen uso de las tecnologías web para desarrollar aplicaciones para dispositivos concretos, denominándose aplicaciones híbridas aquellas que pueden funcionar en diferentes plataformas. Los bancos, las plataformas de vídeo, las empresas de comida para llevar o la gestión de servidores empresariales entre otras muchas acciones se realizan utilizando aplicaciones web y lo que es más importante accediendo a datos sensibles como son los números de cuenta bancaria o tarjetas de crédito.

A la hora de desarrollar una aplicación web es necesario garantizar unos niveles de seguridad mínimos que dependen del tipo de aplicación y sector que permitan su utilización. El nivel de seguridad es un elemento difícilmente medible, pero tal y como se trató en el tema anterior es posible certificar el nivel que una aplicación alcanza en este caso seguridad.

Se hace imprescindible introducir en la fase de desarrollo del software, en este caso la web elementos que permitan comprobar cómo de segura es la aplicación que se está desarrollando de forma que a la finalización de la misma se pueda al menos garantizar su buen funcionamiento ante cierto tipo de ataques y la no existencia de algunas vulnerabilidades. Esta integración se fundamenta en dos elementos:

  • Casos de prueba: Se han de definir, implementar, probar y documentar las diferentes pruebas que se realiza a la aplicación web, por ejemplo que no se envían datos sensibles por “querystring” o que no se exponen servicios web públicamente cuando su uso se debe de realizar una vez autenticado. La superación de los casos de prueba permiten medir lo segura o insegura que es una aplicación.
  • Conocimiento técnico de las aplicaciones web: Es necesario tener conocimientos técnicos del funcionamiento de las tecnologías, los ataques y las vulnerabilidades para  codificar la aplicación de forma que cumpla con el nivel de seguridad requerido.

El tema anterior se basó en el conocimiento de determinación de cómo de segura es una aplicación, es decir a conocer el nivel de seguridad, en concreto con el estándar marcado por la fundación OWASP. Este tema se centra en poder diseñar, implementar, modificar y mantener código y configurar los elementos que intervienen en la aplicación como servidores de forma que puedan cumplir con los niveles establecidos en el estándar para aplicaciones web.

Indicar el desarrollo seguro de aplicaciones web es uno de los aspectos que más conocimientos técnicos necesita en el desarrollo software actual, es necesario tener conocimientos avanzados de:

  • Protocolo HTTP.
  • HTML y CSS
  • Lenguajes de cliente, principalmente Javascript, aunque en framework más reciente también Typescript.
  • Programación asíncrona.
  • Lenguajes de servidor como PHP, javascript del lado del servidor como node.js o python.
  • Entornos de trabajo existentes en cada uno de los lenguajes y patrones de diseño utilizado (MVC, MVVC…) como Laravel, CakePHP en PHP, Django en Python del lado del servidor o Vue y Angular en el lado del cliente.
  • Conocimientos avanzados de SQL.
  • Configuración avanzada de servidores web.
  • Uso de gran variedad de herramientas informáticas como curl, postman o herramienta desarrollador de navegadores.

Estos requerimientos implica la necesidad de disponer de personal especializado en los puntos anteriores, siendo complicado que una única persona pueda desarrollar una aplicación completa en un tiempo razonable con un nivel aceptable de seguridad, estableciéndose en el mercado laboral diversos perfiles técnicos especializados en los elementos anteriores, principalmente:

  • Desarrollador back-end: Encargado de la arquitectura y código del servidor. Dependiendo del tamaño de la aplicación también puede ser el responsable de la gestión de la base de datos.
  • Desarrollador front-end: Su función principal es la del desarrollo de la parte gráfica en el cliente, aunque también incluye cierta lógica de la aplicación como la validación de formularios.
  • Administrador de base de datos: Gestiona el almacenamiento de los datos de la aplicación, optimizando la misma, creando procedimientos almacenados o gestionando permisos y usuarios entre otros.
  • Administrador de sistemas: Se encarga de la configuración de la infraestructura necesaria para la ejecución de back-end como es el servidor web, las copias de seguridad, el nivel de tolerancia a fallos o la escalabilidad de la aplicación del lado del servidor.
  • Testeador: Encargado de diseñar, implementar y probar los casos de prueba y el software, colabora con los desarrolladores como con los administradores.

Existen otros perfiles como analistas, jefes de proyecto, desarrolladores de interfaces o expertos en SEO, la existencia de estos perfiles depende del tamaño de la aplicación, aunque los primeros son los más relacionados con el desarrollo seguro.

Modelo, vista, controlador: Patrón software que separa la aplicación en Modelo, vista y controlador, muy utilizado en el desarrollo web. MVVM: modelo–vista–modelo de vista, variación del MVC que intentan independizar aún más la presentación del resto del elementos de la aplicación

Profesionales que empleando diferentes técnicas permiten mejorar el posiciomiento en los buscadores, en especial Google.
