3.6. -Configuración de seguridad incorrecta.

Suele ser habitual que los piratas informáticos aprovechan configuraciones de seguridad incorrectas. Una configuración errónea de seguridad surge cuando se mantienen con valores predeterminados.

Una aplicación puede ser vulnerable si:

  • El manejo de errores proporciona a los usuarios trazas o mensajes de error demasiado informativos.
  • Las cuentas predeterminadas y sus contraseñas siguen activas y sin cambios.
  • Se encuentran instaladas o habilitadas características innecesarias (ej. puertos, servicios, páginas, cuentas o permisos).
  • Para los sistemas actualizados, las nuevas funciones de seguridad se encuentran desactivadas o no se encuentran configuradas de forma adecuada o segura.
  • El servidor no envía directrices o cabeceras de seguridad a los clientes o se encuentran configurados con valores inseguros.
  • El software se encuentra desactualizado o presenta vulnerabilidades.

Las principales recomendaciones para evitar ataques que aprovechan estas vulnerabilidades son:

  • Uso de una plataforma minimalista sin funcionalidades innecesarias, componentes, documentación o ejemplos. Se debe evitar la instalación de frameworks y funcionalidades no utilizadas.
  • Siga un proceso para revisar y actualizar las configuraciones apropiadas de acuerdo a las advertencias de seguridad y siga un proceso de gestión de parches.
  • Las aplicaciones deben tener una arquitectura segmentada que proporcione una separación efectiva y segura entre componentes y acceso a terceros, contenedores o grupos de seguridad en la nube (ACLs).
  • Se recomienda la utilización de procesos automatizados para verificar la efectividad de los ajustes y configuraciones.
