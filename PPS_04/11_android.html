1.1.- Android.

Los permisos en Android han cambiado a medida que el malware se hacía cada vez más peligroso, y no solo el malware, sino aplicaciones como Uber recolectan información del usuario en segundo plano. En cada nueva versión se ha añadido una granularidad mayor. Un gran salto se produjo en el 2015 con la versión Marshmallow, que introdujo los permisos en tiempo de ejecución, hasta ese momento los permisos se concedían o no, independientemente del momento.

Actualmente Android se encuentra en la versión 11, y siguiendo las versiones anteriores añade nuevas funcionalidades a la gestión de permisos.

Los permisos se definen a nivel de recurso, el sistema administra una lista de permisos asociada a cada recurso a gestionar con las aplicaciones que lo pueden usar, estos se dividen en:

  • Datos restringidos, como la lista de contactos del usuario.
  • Acciones restringidas, como hacer una fotografía desde la cámara.

Se detalla con mayor precisión cada uno de los recursos, acciones y momentos de los permisos

Se establecen diferentes tipos de permisos en función de las necesidades:

Permisos en la instalación

Se otorgan acceso limitado a los datos restringidos y acciones que no afectan al sistema u otras aplicaciones. Dentro de estos permisos se tienen:

  • Permisos normales: Acceso a los datos y acciones más allá de la zona de pruebas. Por ejemplo consultar el estado de la red ACCESS_NETWORK_STATE o BLUETOOTH para permitir a la aplicación conectarse a dispositivos emparejados.
  • Permisos de firma: Si una aplicación posee un permiso, este se puede extender a otra aplicación siempre que la firma digital de ambas sea la misma (mismo desarrollador, empresa..). Alguno de los permisos: DELETE_CACHE_FILES o MANAGE_EXTERNAL_STORAGE.

    Permisos en tiempo de ejecución

    Da acceso a datos restringidos y acciones que afectan de forma más considerable al sistema u otras aplicaciones. Cada vez que se accede a dichos recursos se solicita permisos al usuario. Ejemplos de estos permisos: READ_SMS o RECORD_AUDIO.

    Permisos especiales

    Los más restrictivos sólo la plataforma (Android y por tanto Google) y los OEM (Samsung, Xiaomi…). Estos permisos pueden ser personalizados por el usuario y cada uno de ellos posee sus características propias. Por ejemplo: SYSTEM_ALERT_WINDOW y MANAGE_EXTERNAL_STORAGE.

    Los anteriores permisos se definen en el fichero AndroidManifest.xml que define las características y detalla la configuración de una aplicación para el sistema operativo Android.

    Ejemplo de sección de permisos:
    <manifest ... >

    <uses-permission android:name="android.permission.SEND_SMS"/>

    ...

</manifest>

    Lo tratado anteriormente es como las aplicaciones interactúan con el sistema, pero Android permite que las aplicaciones interactúen entre ellas, pudiendo delimitar la “activity”, el uso de servicios (concepto similar a los demonios en Linux) o proveedores de contenido (elemento que gestiona un conjunto de datos o información). 

    El sistema operativo define un sistema similar a los eventos para que el sistema operativo y las aplicaciones puedan comunicarse de forma asíncrona. Por ejemplo, resulta interesante conocer cuando en sistema pasa a “modo avión” de forma que la aplicación lo tenga en cuenta, y no mande mensajes a través de la red ya que no se encuentra activa, siendo posible definir a qué aplicaciones se envía el evento.  

    Para definir estos permisos entre aplicaciones, también se definen en el fichero AndroidManifest.xml pero en sección dedicada a cada uno de los elementos, <activity><service>....
