1.1.3.- Estableciendo permisos

En el punto anterior se han tratado la solicitud de permisos de las aplicaciones al sistema, que protegen al mismo de acciones de las aplicaciones no deseadas, pero es necesario proteger también las aplicaciones desarrolladas.

Android se basa en parte en la colaboración entre aplicaciones para implementar diferentes funcionalidades, por ejemplo cuando se desea enviar una captura de pantalla se ofrece una lista de aplicaciones de mensajería que se puede utilizar.

La aplicación puede limitar la interacción con otras aplicaciones a nivel de:

  • Actividades. Se puede fijar el usuario (app) a la que se permite iniciar la actividad.
  • Servicios. Componente de una aplicación que puede realizar operaciones de larga ejecución en segundo plano y que no proporciona una interfaz de usuario.
  • Proveedores de contenido. Un proveedor de contenido administra el acceso a un repositorio central de datos. Se encuentran orientados a que lo usen otras aplicaciones que acceden al proveedor. Se ofrece una interfaz estándar y uniforme para los datos.
  • Receptores de emisión. Las apps de Android pueden enviar o recibir mensajes de emisión desde el sistema de Android y otras apps para Android, estas emisiones se envían cuando ocurre un evento de interés

En el fichero Androidmanifest.xml se declaran los anteriores elementos con diferentes etiquetas como <activity><service><provider> y <receiver> y dentro de cada uno de ellos se indican las restricciones.

Entrar en detalle en cada uno de los componentes anteriores sobrepasa la asignación temporal al tema, además el objetivo del mismo no es desarrollar aplicaciones en Android sino conocer y aplicar mecanismos de seguridad.
