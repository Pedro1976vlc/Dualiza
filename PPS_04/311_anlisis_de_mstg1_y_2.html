3.1.1.- Análisis de MSTG-1 y 2

Recordar que estos dos casos de prueba se refieren a el uso de los sistemas de almacenamiento de credenciales para almacenar datos confidenciales, no guardar nunca datos confidenciales fuera del contenedor de la aplicación del sistema de almacenamiento de claves, en este caso Android.

Se establece un análisis estático en el que se compruebas sobre el código elementos como:

  • Fichero AndroidManifest.xml y los permisos solicitados con respecto al almacenamiento.
  • Comprobar  que se establece los permisos de los ficheros como MODE_PRIVATE de forma general y otros de forma excepcional con causa justificada.
  • Revisar el uso correcto de las siguientes clases y funciones relacionadas con el almacenamiento: SharedPreferences, FileOutPutStream, getExternal, getWritableDatabase, getReadableDatabase, getCacheDir y getExternalCacheDirs
  • Usar métodos y algoritmos de codificación fuertes y si es posible utilizando librerías de terceros seguras y ampliamente probadas.

También es necesario un análisis dinámico en tiempo de ejecución:

  • Comprobar los ficheros creados por la aplicación tanto en el almacenamiento interno como el externo, en caso de que se creen comprobar que no se guarda información confidencial en texto claro.
  • Comprobar los ficheros de apoyo en el desarrollo, las copias de seguridad y ficheros antiguos antes de llevar la aplicación a producción.
  • Comprobar la existencia de la base de datos SQLite, que información almacena y si esta es confidencial y está disponible.
  • Asegurarse de que las bases de datos SQLite se encuentran codificadas y protegidas con contraseña y estas se han almacenado en lugar seguro como KeyStore.
  • Comprobar en caso de existir fichero de preferencias (Shared Preferences) no contiene información sensible no protegida.
  • Validar los permisos de los ficheros almacenados en /data/data/<package-name>, solo el usuario y grupo que instaló la aplicación puede leer, escibir y ejecutar (rwx).
  • Examinar si se hace uso de base de datos Firebase Real-time revisando si la configuración es correcta, realizando una llamada HTTP a la URL:

https://_firebaseProjectName_.firebaseio.com/.json

  • Determinar si existen base de datos Realm en /data/data/<package-name>/files/, si no está codificada y posee información sensible.
