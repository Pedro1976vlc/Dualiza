Glosario

Glosario

  • API REST. Una API es un conjunto de definiciones y protocolos que se utiliza para desarrollar e integrar aplicaciones software. Una API de RESTful, es una interfaz de programación que se ajusta a los límites de la arquitectura REST y permite la interacción con los servicios web de RESTful.
  • Disponibilidad. Capacidad de ofrecer los datos en cualquier lugar y en cualquier momento.
  • Elasticidad. Capacidad del sistema para crecer.
  • Escalabilidad. Velocidad de crecimiento de los recursos en función de la demanda.
  • Eavesdropping. Escuchar conversaciones privadas sin consentimiento de forma ilegal.
  • Frida. Framework para inyección de código en aplicaciones nativas de Wiindwos,macOS, Linux, Andrody y QNX
  • Granularildad. Se refiere al nivel de detalle  con el que se describen cada uno de los recursos, acciones y momentos de los permisos.
  • Intent. Una intención (intent) en el sistema operativo Android es un mecanismo software de coordinación entre las diferentes actividades que conforman una tarea mediante la cual se puede solicitar una acción de otro componente de una app.
  • IPC (Inter-Process Communication). Sistema de comunicación entre proceso, existen diferentes tipos:variables compartidas, memoria principal o paso de mensajes entre otros.
  • MASVS (Mobile Application Security Verification Standard). El Estándar OWASP de Verificación de Seguridad de Aplicaciones Móviles, es un esfuerzo comunitario para establecer un marco de requisitos de seguridad necesarios para diseñar, desarrollar y probar aplicaciones móviles seguras en iOS y Android.
  • MSTG (Mobile Security Testing Guide). Guía OWASP de pruebas de seguridad para móviles. Se trata de un documento que  aporta información específica y técnica sobre los controles de seguridad que se deben contemplar a la hora de realizar tareas de pentesting en aplicaciones móviles.
  • Magisk. Herramienta para modificar móviles Android, mediante la cual es posible obtener acceso root en un terminal, descargar e instalar módulos con modificaciones preparados para ser usados.
  • No repudio. En seguridad informática cualidad de un documento que identifica al autor de forma inequívoca y de cuya autoría no puede renunciar.
  • Rooteado. Dispositivo en el cual el usuario del mismo posee permisos de superusuario en el sistema operativo, pudiendo tener acceso a todas las funciones. 
  • SSE (Server-Sent Events). Protocolo que describe los mecanismos necesarios para enviar información asíncrona desde el servidor a los clientes. Formando parte del estándar HTML5 del W3C.
  • Xposed. Framework para modificar el comportamiento del sistema Android y sus respectivas aplicaciones sin realizar modificaciones directas sobre éstas.
