1.1.2.1.- Permisos de instalación o normales

Un ejemplo sencillo en el que se comprueba permisos en tiempo de ejecución y permisos de instalación:

Se desea consultar una API Rest pública del juego Age Of Empires II https://age-of-empires-2-api.herokuapp.com/docs/ y para ello la aplicación necesita acceder a Internet, el permiso necesario es: Internet. https://developer.android.com/reference/android/Manifest.permission#INTERNET

Se modifica el fichero AndroidManifest.xml para añadir la solicitud del permiso.

<?xml version="1.0" encoding="utf-8"?>

<manifest xmlns:android="http://schemas.android.com/apk/res/android"

   package="com.example.myapplication">

   

<uses-permission android:name="android.permission.INTERNET"></uses-permission>

   <application

       android:allowBackup="true"

       android:icon="@mipmap/ic_launcher"

       android:label="@string/app_name"

       android:roundIcon="@mipmap/ic_launcher_round"

       android:supportsRtl="true"

       android:theme="@style/Theme.MyApplication">

       <activity android:name=".MainActivity">

           <intent-filter>

               <action android:name="android.intent.action.MAIN" />



               <category android:name="android.intent.category.LAUNCHER" />

           </intent-filter>

       </activity>

   </application>



</manifest>


Al ser un permiso permanente se puede utilizar sin limitaciones. 

A continuación simplemente utilizar el recurso, se utiliza la librería Volley para realizar las peticiones HTTP

val queue = Volley.newRequestQueue(this)

        val stringRequest = StringRequest(

            Request.Method.GET, url,

            { response ->

                findViewById<TextView>(R.id.visor).setText(response)

            },

            { error ->

                run {

                    Toast.makeText(applicationContext, error.message, Toast.LENGTH_SHORT)

                        .show()

                }

            })

        queue.add(stringRequest)

Si se comenta el permiso en el fichero Androidmanifest.xml al ejecutar el código se obtiene el error:

Java.lang.SecurityException: Permission denied (missing INTERNET permision?)
