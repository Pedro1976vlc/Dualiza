7. -Seguridad en los lenguajes de programación y sus entornos de ejecución (“sandboxes”).

Los lenguajes de programación son la base sobre los que se construyen los sistemas informáticos, desde sistemas operativos generalistas hasta sistemas operativos en tiempo real pasando por software de contabilidad o de entretenimiento. En cuanto a la seguridad de los lenguajes se puede ver desde 2 puntos de vista:

Seguridad en el desarrollo del código. Algunos lenguajes por su características dejan en manos del desarrollador ciertos aspectos que de no realizarse de forma precisa pueden dar lugar al mal funcionamiento del software, uno de los clásicos es olvidar colocar el "break" en algún caso de la sentencia "switch" y ejecutar más instrucciones de las deseadas, otros ejemplos son la no inicialización de variables o inicializarlas a null que pueden ser un problema en la posterior ejecución del código, el uso y conversión de tipos de datos, o el uso de punteros en C/ C++ aunque este último caso a pesar de poder ser un problema en la ejecución del programa y en la fuga de memoria, permite una gran flexibilidad, expresividad y velocidad de ejecución. En la actualidad los compiladores permiten configurar el grado de análisis y cumplimiento de ciertas reglas que fomentan la creación de código en teoría seguro, por ejemplo Typescript o C# permiten definir un modo estricto de generación.  

Entre los lenguajes de programación más seguros en cuanto a su compilación destacan:

  • Ada: Basado en Pascal, se caracteriza por ser multiproposito, orientado a objetos y concurrente, el tipado el fuerte, por lo que lo hacen un candidato para los sistemas en tiempo real y críticos por poder gestionar hijos y procesos en paralelo, además sus estructuras minimizan los posibles errores, por ejemplo la instrucción switch.
Do_Menu_Choices_1:
   loop
      ...
  
      case User_Choice is
         when 'A'    => Item := Terminal_IO.Get ("Item to add");
         when 'D'    => Item := Terminal_IO.Get ("Item to delete");
         when 'M'    => Item := Terminal_IO.Get ("Item to modify");
         when 'Q'    => exit Do_Menu_Choices_1;
  
         when others => -- error has already been signaled to user
            null;
      end case;
   end loop Do_Menu_Choices_1;
  • Rust: Lenguaje que ha aparecido recientemente de la mano de la fundación Mozilla y que está experimentando un gran interés. Se basa en C++ mejorando entre otros la gestión de la memoria, la existencia de un sistema de conversión de tipos que facilita el polimorfismo, además implementa un sistema de concurrencia optimizado basado en el paso de mensajes.

Seguridad de la plataforma: Una vez el código compilado no se tiene la seguridad de que instrucciones está ejecutando o si el autor de esa librería o módulo es confiable. Para poder ejecutar código seguro los diferentes lenguajes y entornos proporcionan herramientas para garantizar la autoría del código, principalmente firmando los ejecutables con certificados digitales. Uno de los primeros entornos que introdujeron la necesidad de firma para la ejecución de código fue Java con su tecnología Java Web Start que permitía la ejecución. Otros optan por realizar funciones resumen o hash del código original como Javascript y el atributo "integrity" de la etiqueta "script", junto con el atributo "crossorigin" para indicar la url de los lugares permitidos.
