Ejercicio Resuelto

Docker-Compose y Wordpress:

A partir del siguiente fichero, disponible en la página oficial de Docker (Docker.com) crear el fichero docker-compose.yaml  y realizar las tareas que se indican a continuación:

version: "3.9"
    
services:
  db:
    image: mysql:5.7
    volumes:
      - db_data:/var/lib/mysql
    restart: always
    environment:
      MYSQL_ROOT_PASSWORD: somewordpress
      MYSQL_DATABASE: wordpress
      MYSQL_USER: wordpress
      MYSQL_PASSWORD: wordpress
    
  wordpress:
    depends_on:
      - db
    image: wordpress:latest
    volumes:
      - wordpress_data:/var/www/html
    ports:
      - "8000:80"
    restart: always
    environment:
      WORDPRESS_DB_HOST: db:3306
      WORDPRESS_DB_USER: wordpress
      WORDPRESS_DB_PASSWORD: wordpress
      WORDPRESS_DB_NAME: wordpress
volumes:
  db_data: {}
  wordpress_data: {}

Cuestiones:

Indicar cuantos servicios se levantan. 

Retroalimentación

Dos servicios, la base de datos y el servidor web que contiene Wordpress.

¿Qué imágenes descarga?

Retroalimentación

wordpress:latest y mysql:5.7.

¿Qué servicio se inicia antes?. Indicar la razón.

Retroalimentación

Se inicia antes la base de datos ya que el servidor web depende de esta según propiedad depend_on.

¿En qué puerto local se encuentra escuchando el servidor web?. Para acceder a la web ¿Qué dirección y puerto se han de poner en el navegador?

Retroalimentación

El puerto es el 80, pero desde el exterior se accede por el 8000 ya que se redirige según el fichero de configuración, la dirección IP será la de la máquina real.

¿Cuántos volúmenes se crean? ¿Qué sucede con los volúmenes si se borran los contenedores?.Si se desea realizar una copia de seguridad de los volúmenes. ¿Dónde se encuentran? 

Retroalimentación

Se crean dos volúmenes. Si se borran los contenedores los volúmenes siguen existiendo. Si se desea realizar una copia de seguridad de los volúmenes, estos se encuentran en /var/lib/docker.

Instalar Docker-Compose:

Crear un directorio para contener el fichero docker-compose.yaml y ejecutar el comando:

docker-compose up -d

Retroalimentación

¿Qué función realiza el parámetro -d?

Retroalimentación

El parámetro -d lanza los servicios en segundo plano.
