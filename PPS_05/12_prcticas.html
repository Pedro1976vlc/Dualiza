1.2.- Prácticas

El ingeniero de DevOps introduce procesos, herramientas y metodologías para equilibrar las necesidades durante todo el ciclo de vida del desarrollo de software, desde la codificación y la implementación hasta el mantenimiento y las actualizaciones.

El objetivo de DevOps es la unificación y la automatización de procesos, y los ingenieros de DevOps son fundamentales para las tareas relacionadas con la combinación de código y el mantenimiento y la gestión de aplicaciones.

Además de definir las fases se establecen una serie de prácticas que los equipos DevOps deben de implementar:

  • Integración continua: Existe un repositorio de código en el que se van añadiendo el nuevo código, creándose las versiones, realizando las pruebas  y si son satisfactorias pasando a la siguiente fase, este proceso se ha de automatizar lo máximo posible.
  • Entrega continua. Relacionada con la anterior, consiste en compilar, probar y preparar los cambios de código para la fase  de configuración y/o despliegue. De igual forma ha de ser lo más automatizada posible.
  • Microservicios. Filosofía en el que una aplicación es un conjunto de servicios pequeños y estos se ejecutan en un proceso independiente comunicándose con un protocolo sencillo (HTTP usualmente). Cada microservicio posee un único propósito y se pueden utilizar diferentes lenguajes ya que posee una API para comunicarse.
  • Infraestructura como código. La virtualización y los servicios en la nube permiten adaptar la infraestructura a las necesidades de uso en cada momento, estos ajustes pueden realizarse de forma manual o automática, por ejemplo ante un pico de demanda. En el segundo caso la interacción con la infraestructura se trata como código, siguiendo el caso anterior, crear de forma automática nuevos servidores para responder al pico a partir de plantillas existentes.
  • Administración de la configuración. La infraestructura como los sistemas operativos o servidores  y las aplicaciones necesitan ser configuradas. Usando la misma filosofía del punto anterior, dichas configuraciones se han de realizar de forma automatizada y usando código, minimizando la interacción de los desarrolladores y administradores.
  • Política como código. Las empresas suelen definir políticas y requerimientos para su infraestructura, por ejemplo configuración de seguridad o criterios de contraseñas. Si estas políticas se implementan usando código son más fácilmente desplegables además de poder verificar su cumplimiento verificando la validez del código.
  • Monitoreo y registro. Las organizaciones almacenan y evalúan la información producida tanto por la aplicación como por la infraestructura para la mejora del producto a través de métricas. Esta tarea permite a partir de la información obtenida en tiempo real o casi, actuar ante posibles alertas o prever posibles necesidades.
  • Comunicación y colaboración. Una de las prácticas que determinan en mayor medida el éxito de la aplicación de DevOps, se han de definir de forma clara los flujos de trabajo y las responsabilidades de los equipos de desarrollo y operaciones, compartiendo información en un proceso de mejora continua.
