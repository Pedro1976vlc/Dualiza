2.1.6.- Hook / Ganchos

La gestión de código con repositorios puede acarrear además una serie de tareas como por ejemplo pasar un conjunto de pruebas previas antes de fusionar un feature con una release. Esta tarea puede hacerse de forma manual o crear guiones que ante eventos del repositorio git se ejecuten.

Los guiones se encuentran en el directorio .git/hooks, se proporcionan ejemplos con la extensión .sample para que no se ejecuten, aunque cada desarrollador puede modificar estos o añadir nuevos. Un ejemplo del  guión commit-msg.sample:

Los guiones se escriben principalmente utilizando shell o PERL, aunque se puede usar otro lenguaje de guiones indicando al inicio del fichero.

Algunas de las acciones y ganchos que se ejecutan son:

  • Pre-commit. Se realiza previamente al realizar un commit, un uso clásico es la ejecución de un conjunto de pruebas que se han de pasar para realizar la confirmación.
  • Prepare-commit.msg. Previo al inicio de editor de texto para añadir el mensaje de confirmación. No posee mucha utilidad.
  • Commit-msg. Recibe un parámetro, la ubicación del archivo temporal que contiene el mensaje de confirmación actual. Útil para establecer patrones de mensajes de confirmaciones.
  • Post.commit. Se suele utilizar para realizar notificaciones o tareas similares.
  • Post-checkout. Una vez se ha cambiado el commit, úilt para generar documentación de forma automática.
  • Post-merge. Al realizarse una fusión.
  • Pre-push. Antes de transferir los objetos al servidor remoto, pero cuando se ha enviado la referencia.
