Bibliografía

Fuentes documentales utilizadas en esta unidad

  • Cultura DevOps. Uno de los mejores blogs de Internet para la difusión de la cultura DevOps en español. Incorpora información y vídeos sobre la metodología y herramientas que los ingenieros DevOps deben conocer para introducirse con éxito en la cultura DevOps. https://culturadevops.blogspot.com
  • Guía completa IBM Cloud Learn Hub - What is DevOps? La multinacional IBM dispone de su propia guía para el aprendizaje de DevOps. https://www.ibm.com/cloud/learn/devops-a-complete-guide
  • Página del proyecto Git. Contiene la documentación oficial, manuales y el software básico. https://git-scm.com/
  • Web del proyecto Gradle. Interesante la sección de entrenamiento de la misma, con cursos sobre el uso de la herramienta. https://gradle.org/
  • Documentación oficial del software de virtualización de contenedores Docker: https://docs.docker.com/, destacar la sección de Docker-Compose: https://docs.docker.com/compose/ y la referencia del fichero Dockerfile https://docs.docker.com/engine/reference/builder/.
  • Web con la documentación del proyecto Kubernetes. Sistema de código libre para la automatización del despliegue, ajuste de escala y manejo de aplicaciones en contenedores​. Actualmente, Cloud Native Computing Foundation se encarga de su gestión y documentación. https://kubernetes.io/es/docs/home/
  • Proyecto Minikube basado en Kubernetes. Resulta muy interesante para iniciarse de forma sencilla en Kubernetes: https://minikube.sigs.k8s.io/
