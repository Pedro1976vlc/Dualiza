Introducción

El desarrollo del software ha evolucionado desde sus inicios hasta la actualidad, convirtiéndose casi en un procedimiento de una fábrica de la era industrial, automatizando la mayor parte de los procesos en busca de:

  1. Mejora del código fuente. Existe diferente software en el que se le puede indicar reglas de escritura del código, por ejemplo Checkstyle para Java, PHP_CodeSniffer para PHP o  ESLint para Javascript.
  2. Obtener código libre de fallos. Si bien es un objetivo difícil de cumplir al 100%, el software tratado en el tema 1 para los casos de prueba garantiza un software al menos evaluado para los casos de prueba.
  3. Conseguir software seguro. Conseguir software seguro al 100% es muy complicado, pero cumpliendo con el estándar OWASP tratados en las unidades 2,3 y 4 se puede conseguir software con un nivel de seguridad más que aceptable.
  4. Minimizar los tiempos de desarrollo. Todas las tareas anteriores realizadas de forma independiente, manual y no integrada implican una pérdida de tiempo considerable, además de introducir el factor humano y por tanto también mayor probabilidad de error. Entre los objetivos de esta unidad se encuentra la de minimizar y automatizar entre otras las tareas anteriores.
  5. Construir y desplegar aplicaciones de forma automatizada. Esta puede ser una tarea relativamente compleja y costosa en tiempo, en esta unidad se tratan algunas de las herramientas de despliegue más utilizadas en los diferentes lenguajes.

Pero un sistema no solo depende del desarrollo y de la aplicación propiamente dicha, y menos mucho menos en la actualidad, se necesita una infraestructura que de un soporte flexible a la aplicación. Disponer de una infraestructura segura y escalable es tan importante como la seguridad de la propia aplicación. En este tema se tratan las principales tecnologías que permiten crear una infraestructura, centrándose en la virtualización de sistemas.

Por último, es necesario establecer mecanismos de gestión en caso de que se produzcan fallos, como puede ser la caída de un servidor a través de herramientas que respondan de forma automatizada a dichos fallos para que el sistema se recupere lo antes posible.
