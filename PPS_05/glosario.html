Glosario

Glosario

  • Apache Subversion. Aplicación de software libre para el control de versiones, basada en un repositorio cuyo funcionamiento es muy similar al de un sistema de ficheros.
  • Cluster. Conjunto de computadoras unidas entre sí y que desde el exterior se tratan como un único sistema.
  • CVS (Concurrent Versions System). Aplicación informática que implementa un sistema de control de versiones: mantiene el registro de todo el trabajo y los cambios en los ficheros. Permite que distintos desarrolladores colaboren, sin importar que estos se encuentren separados por grandes distancias.
  • Integración continua (CI). Es la práctica en la que las versiones de código son continuamente integrados en un solo repositorio. Así el nuevo código es testeado de manera inmediata y se corrigen errores más rápido.
  • Implementación continua (CD). Es crear un flujo completo para la entrega paquetes de código para su inmediata implementación.
  • Kubernetes (K8s). Es una plataforma de código abierto para automatizar la implementación, el escalado y la administración de aplicaciones en contenedores.
  • Metodologías ágiles. Conjunto de metodologías de la ingeniería software basadas en el desarrollo iterativo e incremental, que buscan reducir la sobrecarga de trabajo en el proceso de obtención de nuevas aplicaciones. Los requisitos y soluciones evolucionan con el tiempo según la necesidad del proyecto adaptando las formas de trabajo, incrementando la flexibilidad e inmediatez en la toma de decisiones. Ejemplos: SCRUM, XP(Programación extrema), Lean, etc.
  • SDL. Lenguaje de programación en un ámbito de aplicación específico.
  • Volumen. Unidad lógica de almacenamiento asignada a un tipo de sistema de archivos.
  • YAML (YAML Ain't Markup Language). Es un lenguaje de serialización de datos similar a XML y JSON,. Es muy útil para las necesidades de programación que van desde archivos de configuración hasta mensajería de Internet, persistencia de objetos y auditoría de datos.
