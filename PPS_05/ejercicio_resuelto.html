Ejercicio resuelto

Ejercicio Resuelto

Instalación de una imagen del servidor nginx.

Se desea crear un contenedor para dar soporte al desarrollo de una aplicación web realizada con PHP.

Se busca en el repositorio una imagen que soporte PHP, en este caso webdevops/php-nginx con un servidor nginx.

Retroalimentación

Se descarga la imagen:

docker pull webdevops/php-nginx

Y viendo como se encuentra en local.

docker image ls

Retroalimentación

Se instancia el contenedor con:

docker run  -p 80:80 webdevops/php-nginx

Y  desde el navegador web, entrando en la URL se tiene:

Retroalimentación

Ahora es necesario copiar los ficheros de la web al contenedor con el comando cp, en la carpeta tempo se encuentra un fichero HTML sencillo llamado index.html:

docker cp ./tempo affectionate_davinci:./app

 

Al entrar en la web se tiene:

Retroalimentación

Los contenedores son elementos no persistentes, que pueden ser eliminados en cualquier momento, además si se levanta otro contenedor a partir de la imagen, esta no contendrá los datos del contenedor anterior. La solución es usar volúmenes, de forma que se puedan compartir entre contenedores. Se crea un simple fichero PHP que almacena datos de la petición en formato csv (podrían ser ficheros de configuración, base de datos…).

Para crear un volumen ejecutar:

Retroalimentación

docker volume create  datos

Instanciar un nuevo contenedor a partir de la imagen, indicando que el volumen a usar con la ruta de anfitrión y la ruta que tendrá en el contenedor:

 

Retroalimentación

docker run  -p 81:80 -v datos:/datos webdevops/php-nginx &



Se copian los ficheros de la aplicación de igual forma que el anterior contenedor (obtener el nombre del contenedor con docker container ls):

Retroalimentación

docker cp ./tempo/index.php interesting_zhukovsky:./app

El código PHP es sencillo:

Retroalimentación

<?php
$fp = fopen('/datos/datos.csv', 'a+');
fwrite($fp,  "'".date("Y-m-d H:i:s")."';'".$_SERVER['REMOTE_ADDR']."';'".$_SERVER['HTTP_USER_AGENT']."'\n");
fclose($fp);


echo "Almacenando los datos en volumen /datos";
?>

Se puede ver el fichero entrando en el contenedor:

docker exec -it 8622274c5576 bash


Los volúmenes son almacenados en el anfitrión en:

/var/lib/docker
